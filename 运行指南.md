# EchoMonitor 运行指南

## 环境要求

- Android Studio Hedgehog (2023.1.1) 或更高版本
- JDK 17
- Android SDK 34
- NDK 25.1.8937393
- CMake 3.22.1

## 运行步骤

### 1. 打开项目

```bash
# 在Android Studio中打开项目
File → Open → 选择 EchoMonitor 文件夹
```

### 2. 同步Gradle

首次打开会自动同步，或点击：
```
File → Sync Project with Gradle Files
```

### 3. 安装依赖

Android Studio会自动下载：
- Gradle 8.2
- Oboe 1.8.1
- AndroidX依赖

### 4. 配置设备

连接Android设备（Android 8.0+）：
```bash
# 检查设备连接
adb devices

# 预期输出：
List of devices attached
xxx1234567890    device
```

### 5. 运行应用

点击 **Run** 按钮（绿色三角形）或按 `Shift + F10`

## 可能遇到的问题

### 1. NDK未找到

**解决**：
```
Tools → SDK Manager → SDK Tools → 勾选 NDK
```

### 2. CMake版本不匹配

**解决**：
```
Tools → SDK Manager → SDK Tools → 勾选 CMake 3.22.1
```

### 3. 权限被拒绝

首次运行需要授予：
- 录音权限（RECORD_AUDIO）
- 忽略电池优化（可选）

### 4. 低延迟路径未启用

检查设备是否支持AAudio：
```kotlin
// 在AudioEngine.kt中添加调试日志
Log.d(TAG, "Using AAudio: ${oboeStream.usesAAudio()}")
```

## 调试技巧

### 查看日志

```bash
adb logcat -s AudioEngine_CPP:D AudioMonitoringService:D EchoMonitor:D
```

### 测量延迟

应用会每2秒输出当前延迟：
```
D/AudioEngine_CPP: Current latency: 15.3ms
```

## 构建命令行

```bash
# 构建Debug版本
./gradlew assembleDebug

# 安装到设备
./gradlew installDebug

# 运行测试
./gradlew test
```
